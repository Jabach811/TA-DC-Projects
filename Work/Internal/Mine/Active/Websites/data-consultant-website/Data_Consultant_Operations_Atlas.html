Preparation

File prep before anything runs.

The strongest prep topics in legacy are validating input data first, naming and folder discipline, trimming the parameter file to only the workflow you need, keeping `WIRE_N` on until ready, and checking source folder paths every run.

Best practices

Patterns that prevent silent damage.

The best-practice lane is practical: match totals first, force numeric fields to General format, use the column-width macro, keep loan files pipe-delimited, and strip mapping files down to debug instead of guessing.

Overview + prep

Build the file right before you submit it.

The prep lane explains what EDS handles, how it differs from Informatica, the required pipe-delimited file format, strict field-order sensitivity, and the pre-submit checklist.

  • Dates: MMDDYYYY
  • Deferrals: decimals like `0.05`
  • Dollars: 2 decimals, no symbols
  • Blank means blank, not `NULL`
  • Pipe-delimited UTF-8 output
