# Patterns

## Brand-New Pages

Good default structure:
- top-level hero with strong positioning
- clear branch or CTA choices
- one or two supporting panels that explain use or scope
- a structured body with grouped sections and labeled inputs/cards

Useful for:
- landing pages
- intake hubs
- service microsites

## Redesign Concepts

When asked for multiple redesigns:
- keep the same content map
- change theme and shell styling
- create separate files
- give each concept a distinct identity

Examples of differences:
- editorial premium
- technical system/grid
- dark glossy ops console
- warm studio aesthetic

## Standalone App Behavior

For local HTML tools:
- open tool files directly in a new tab when app-like independence is desired
- expose copy-link or open-direct controls inside the tool if users may bookmark it

## Checklist / Tracker Dashboards

Useful inputs for dashboards:
- saved plans in localStorage
- current checklist state
- completion percent
- status
- plan type
- feature flags
- phase counts
- remaining tasks

Useful dashboard sections:
- snapshot cards
- plan mix / feature mix
- recent activity
- attention queue
- clickable plan cards back into the app

## Intake Form Content

For web apps, capture:
- goal
- users
- permissions
- workflows
- pages/modules
- data inputs
- integrations
- exports
- saved records
- design direction

For websites, capture:
- business goal
- audience
- main CTA
- number of sections
- number of subsections
- page architecture
- content requirements
- navigation rules
- conversion goals
- aesthetic direction
