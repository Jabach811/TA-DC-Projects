02 Best Practices 07 items · Golden rules for clean runs
01
Keep the Safety On
The test flag (WIRE_N) is like the safety on a gun. Keep it on until you are absolutely ready to fire. Verify this setting every single time — it’s easy to forget when you’re in a rhythm.
02
Match Your Totals First
Validate that ref numbers match dollar amounts before importing — CONV file, balance file, and P2 ref number setup. A mismatch is either an error or a silent failure. Catch it before you run.
03
General Format — Always
Set all numeric columns to General format in Excel before building your input file. Accounting format adds a leading space that drops characters from ref numbers on load. One of the most common silent error causes.
04
Use the Column Width Macro
The CITS balance file requires fixed-width columns. Use the macro — no one should ever set those widths manually. Consistent column widths are required for the file to process correctly.
05
Loans: Pipe-Delimited Only
Loan files must be pipe-delimited text files. CSV (comma-delimited) will fail. This is a common mistake — always convert before importing. Same rule applies to elections.
06
Strip It Down to Debug
Can’t find the issue? Strip the mapping file down to just CIT rows and start eliminating variables one at a time. Add back incrementally until the issue surfaces. Surgical debugging beats guessing.
07
Add Leading Zeros to SSNs
For CIT Balances, participants with fewer than 9 SSN digits generate kickouts. Use =TEXT(A2,"000000000") to pad every SSN to 9 digits for clean output.

Format & Clean Each Field
Dates: MMDDYYYY. Deferrals: decimal (5% = 0.05). Dollars: two decimal places. Status: single uppercase char. Blanks: empty — never NULL, N/A, or zero unless spec says so.
04
Build the Pipe-Delimited File
Export from Excel as .csv, open in Notepad++, find-replace commas with pipes ( | ). Spot-check 5–10 rows. Confirm no stray quotes, no extra commas, no blank trailing rows. Save UTF-8 without BOM.

Row count matches expected · No duplicate SSNs · All dates MMDDYYYY · Deferrals are decimals · Dollars have 2dp · Status codes 1-char uppercase · UTF-8 pipe-delimited · No NULL placeholders.

Date Format · Most Common
Invalid Date Format
ERR
✗ Error: Invalid date format in field [Eligibility Date]
Fix: Ensure all dates are MMDDYYYY with no slashes, dashes, or other separators. In Excel: =TEXT(A2,"MMDDYYYY"). Paste-special values, verify no date formatting remains.
Deferral Rate · Format
Deferral % Not in Decimal Format
ERR
✗ Error: Deferral rate [5] out of range or invalid format
Fix: EDS expects decimals — 5% must be 0.05, not 5. In Excel: =A2/100 then paste-special values. Verify the output shows 0.0500 not 5.0000.
SSN · Lookup
SSN Not Found in System
ERR
✗ Error: SSN [000000000] not found
Fix: Verify the SSN is 9 digits, zero-padded, no dashes. Cross-reference against the enrollment system — participant may not yet be enrolled. If SSN is correct and participant is enrolled, check for leading/trailing spaces in the SSN field.
Column Count · File Structure
Column Count Mismatch
ERR
✗ Error: Expected [X] columns, found [Y] in row [Z]
Fix: Open the file in Notepad++ and count pipe characters on the problem row. An extra or missing pipe = wrong column count. Causes: formula producing a value with a pipe character inside it, extra blank column, missing required field left empty without a pipe placeholder.
Auto-Enroll · Missing Flag
Missing Auto-Enrollment Flag on Auto-Enroll Plan
CAUTION
✗ Participants silently enrolled at wrong deferral rate (or rejected)
Fix: Confirm whether the plan has auto-enrollment via the plan document. If yes, add the auto-enroll flag field (Y), auto-enroll effective date, and default deferral rate. Resubmit. If participants were already silently enrolled at wrong rate, a correction load will be required.
