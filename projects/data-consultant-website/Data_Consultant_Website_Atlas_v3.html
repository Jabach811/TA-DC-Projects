Operating model

Use the portal three different ways depending on the job.

Some work starts with finding the right file or section. Some work starts by launching a checklist or tool. Some work starts by opening a system reference and staying there for a while. The portal has to support all three without making people jump through extra layers.

Start at the homepage, pick the right section, and move into documents, walkthroughs, forms, or reference shelves.

This page now acts as a practical top layer for the DC portal.

What is already here

Section routing, working shelves, tool launches, flow entry points, reference routes, and direct links back into the detailed portal.

What still lives in the old page

The largest detailed content sets, long lists, and full legacy section bodies still live in `v2_10` until they are migrated one by one.

Best next migration target

Documents, Projects, or EDS would be the strongest next full section to port because people use them often and they carry a lot of day-to-day value.

Why this version matters

It gives the team a cleaner first-stop page without taking away the current portal while the migration is still in progress.

Positioning thought
This page should answer one question quickly: where does the work live?
Documents, tools, flows, forms, and reference sections all have a clear place now. That makes the page useful even before every detailed section is fully rebuilt inside Atlas.
