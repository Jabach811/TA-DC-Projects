OVERVIEW
What this workbook does, the problem it solves, and how it's structured
The Problem It Solves
No more manual server switching or ref# typos
Wrong server region (TD vs CoreTV) is invisible until something breaks — and you can lose a lot of time chasing it. Manual ref number entry in queries means trailing spaces and typos that are impossible to see. This workbook eliminates both: server routing is automatic based on case number, and ref numbers are validated and injected cleanly every time.
What It Does
Entry → routing → refresh → summarize → export
  • UserForms for case number and ref number entry — no raw input box
  • Detects server region from case number prefix automatically
  • Supports single ref# (cash) or multiple ref#s (mapping/TIK, one per fund)
  • Injects all parameters into Power Query connections and refreshes
  • Auto-runs summarization by ref#, social, source, and fund
  • One-click copy: table + summary exported together in one paste
Ref Number Rules
Varies by conversion type
  • Cash conversion → 1 ref number total
  • Mapping → 1 ref number per fund
  • Transfer In-Kind → 1 ref number per fund
  • Left sidebar panel handles entry for all scenarios

SETUP GUIDE
How to configure this workbook from scratch
WORKBOOK SETUP
01
Create the left sidebar panel
Reserve a fixed column range on the left of each sheet for the sidebar. Add "Enter Case #" and "Enter Ref #(s)" buttons wired to LaunchCaseEntry and LaunchRefEntry.
02
Add named ranges for sidebar display
Create named ranges (e.g. rng_CurrentCase, rng_RefList) in the sidebar area. UpdateSidebarDisplay() writes to these after every entry.
03
Set up Power Query connections per tab
One connection per extraction table. Leave placeholder values for case #, ref #(s), and server region in each M query. The injection layer replaces these at runtime.
04
Add Copy button to each sheet
Insert a button on every tab wired to CopyTableAndSummary in mod_Export. Position it near the summary area for easy access.
KEY CONFIGURATION POINTS
→
Fill in server region routing logic
In DetectServerRegion() in mod_ParamValidator, add the actual prefix-to-region mapping. This is the critical piece that eliminates the TD/CoreTV switching error.
→
Wire connections to parameters in mod_QueryEngine
In InjectAndRefresh(), add a Select Case block mapping each connection name to the parameters it needs — case #, ref #(s), region, and any dates.
→
Define summary output ranges
In BuildAndWriteSummary(), set the target ranges for each breakdown table. The Copy export in mod_Export relies on knowing where the summary ends.
→
Ref number count — cash vs mapping/TIK
Cash conversions use a single ref number. Mapping and TIK conversions use one ref per fund. The frmRefEntry form handles both — update the list UI to match how many refs your most complex plans will have.
